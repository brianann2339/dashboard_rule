# 戰情室資料解析與圖表計算說明

## 📊 整體數據來源解析

戰情室的數據由 Google Apps Script (`Code.gs`) 在背景定時收集與計算，主要資料來源分為三大類：

- 🟢 **[系統爬蟲擷取] 即時病患資料：** 系統會自動登入並爬取醫院急診網頁（ErBedQry_ER.aspx）的表格內容。會抓取的欄位包含：**狀態**（是否為現掛、看診中）、**檢傷級別**（1~5級）、**掛號時間**、**床位名稱**、**科別**、**動向**（OA/PA/OACU 等）。
- 🟡 **[直接給定數字] 醫護人力班表：** 護理師人數與醫師排班目前是**直接寫死 (Hardcoded) 填寫在程式碼常數中**（包含 `NURSING`, `SCHEDULE`, `NP_SCHEDULE`）。系統會根據執行當下的時間，自動去對應陣列中的當班人數。
- 🟣 **[外部歷史試算表] 歷史滾動數據：** 28天的等床大數據，是透過程式讀取另一份 Google 試算表（`ER_Data` 工作表）中的歷史結案資料。

## 一、關鍵 KPI 與核心指標計算

| 指標名稱 | 資料來源與詳細計算方式 |
| :--- | :--- |
| **戰情痛苦指數**<br>(Pain Index) | **計算邏輯：** 綜合評估急診當下的環境壓力與醫療團隊戰力，計算公式為：<br>`痛苦指數 = 環境壓力(加分) - 團隊戰力(減分)` *(若結果小於0，則以0計)*<br><br>🔴 **環境壓力 (指標加分，數值越高表示急診越緊繃)：**<br>• 過去 1 小時來診人數 × 2<br>• 現掛 (未看診) 人數 × 3<br>• 急救室人數 × 10<br>• 待一般床 (OA/PA) 人數 × 2<br>• 待加護病房 (ICU) 人數 × 5<br>• 無動向人數 × 1<br>• 滯留 > 24 小時人數 × 3<br>• 滯留 > 48 小時人數 × 5<br>• 滯留 > 72 小時人數 × 10<br><br>🟢 **團隊戰力 (指標減分，數值越高表示紓解能力越強)：**<br>• 主治醫師人數 × 10<br>• 住院醫師人數 × 5<br>• 總醫師/高階住院醫師人數 × 10<br>• 專科護理師 (NP) 人數 × 5<br>• 護理師總人數 × 2<br>*(特例：資深主治醫師人數 × 5，此項在程式碼中設定為扣減團隊總戰力)* |
| **醫師人力**<br>(Physician Count) | 🟡 **[直接給定數字]**<br>**資料來源與計算：** 系統會取用當下時間，自動比對 `SCHEDULE` 及 `NP_SCHEDULE` 常數。<br>• **主治醫師：** 預設在 10:00~22:00 時段為 4 人，其餘時段為 3 人。若當班有「兒科支援醫師」，主治總數再 **+1**。<br>• **其他醫師：** 住院醫師、總醫師/資深住院醫師、資深主治醫師，皆依照班表寫入的數字套用。<br>• **專科護理師 (NP)：** 依據白班(08-16)、小夜(16-00)、大夜(00-08) 取值。 |
| **各分區人數判定**<br>(Area Classification) | 🟢 **[系統爬蟲擷取]**<br>**資料來源與計算：** 透過 `classifyPatient()` 讀取每一列的**「床位名稱」**與**「科別」**進行歸類：<br>• **急救室：** 床位包含「急救室」或「急」字開頭。<br>• **婦產科：** 科別包含「婦產」或床位包含「婦」字。<br>• **兒科：** 科別包含「兒」字，或床位以字母 `P` 開頭。<br>• **留觀區：** 床位以字母 `C` 或 `F` 開頭（排除 "C柱下" 與 "F柱下"）。<br>• **A 區：** 床位為「C柱下」，或以 `A`、`1`、「篩」字開頭。若不符其他區，**預設歸類於 A 區**。<br>• **B 區：** 床位為「F柱下」，或以 `B`、`2`、「縫」字開頭。 |

## 二、戰情室圖表計算方式與來源細節

| 位置 | 標題 | 圖表資料來源與詳細計算方式 |
| :--- | :--- | :--- |
| **主頁面**<br>`id="trend"` | **病人數趨勢** | 🟢 **[系統爬蟲擷取]**<br>**資料來源：** 抓取急診網頁的**「狀態」**與**「檢傷級別」**欄位。<br>**計算邏輯：**<br>• `total` (總人數)：狀態包含「看診中」、「現掛」或「回診」即計算在內。<br>• `L1~L5`：判斷「檢傷」欄位，落入 01~05 即加總至該級別。<br>• **圖表呈現：** 遍歷選定的時間區間，分別讀取每個時間點的 `total`、`(L1+L2)`、`(L3+L4+L5)` 以及各別的 `L1` ~ `L5` 組成折線。 |
| **主頁面**<br>`id="ratioChart"` | **護病比趨勢** | 🟡 **[直接給定數字]** 🟢 **[系統爬蟲擷取]**<br>**資料來源：** **護理師人數**從 `NURSING` 常數取得，**病患人數**從網頁抓取並依分區歸類。<br>**計算邏輯：** 每個時間點的護病比皆為 `該區病人數 ÷ 該區護理師人數`。<br>• **總護病比：** 總人數 ÷ 當班總護理師人數。<br>• **護理師分配規則：** 急救室預設 1-2人、留觀區 2人、兒科 1人、檢傷 1人。A區與B區平分剩餘護理師 `(總數 - 5或6) ÷ 2`。 |
| **彈出視窗**<br>點擊「來診」 | **每小時來診量趨勢** | 🟢 **[系統爬蟲擷取]**<br>**資料來源：** 網頁的**「報到/掛號時間」**欄位。<br>**計算邏輯：** 計算 `(現在時間 - 掛號時間)`，若時間差在 0~60 分鐘內，視為「過去 60 分鐘內的來診」，加總後繪製成單一面積折線圖。 |
| **彈出視窗**<br>點擊「待床/待ICU」 | **待床 / 待ICU趨勢** | 🟢 **[系統爬蟲擷取]**<br>**資料來源：** 網頁的**「動向」**欄位。<br>**計算邏輯：**<br>• 若動向含有 `OACU` 或 `PACU`，算作 **待ICU人數**。<br>• 若動向含有 `OA` 或 `PA` (且排除加護病房字眼)，算作 **待一般床人數**。<br>• 繪製成兩條對比折線圖。 |
| **彈出視窗**<br>點擊「等床中位數」 | **現存等床時間分析** | 🟢 **[系統爬蟲擷取]**<br>**資料來源：** 網頁的**「掛號時間」**與**「動向」**欄位。<br>**計算邏輯：** 篩選出動向為「待床」或「待ICU」且非回診病人。計算實際等床時間 `(現在時間 - 掛號時間)`，求出**平均數**、**Q1(25%)**、**中位數**、**Q3(75%)**，除以 60 轉換為小時後繪製成四條線的趨勢圖。 |
| **彈出視窗**<br>點擊「平均待床28天」 | **平均待床 / 平均ICU分析** | 🟣 **[外部歷史試算表]**<br>**資料來源：** 從 `ER_Data` 試算表讀取過去的**「掛號時間」、「滯留時間」、「動向」**欄位。<br>**計算邏輯：** 往前推算 28 天，篩選出歷史動向包含 OA/PA/OACU/PACU 且有「住院」字眼的已結案資料。擷取實際「滯留時間」算出該資料池的平均值、中位數、Q1、Q3 繪製折線圖。 |
| **彈出視窗**<br>點擊「總人數」(上) | **檢傷分佈** | 🟢 **[系統爬蟲擷取]**<br>**計算邏輯：** 抓取系統最新一筆資料快照，將當下 `L1` 到 `L5` 的實際人數，繪製成單一時間點的甜甜圈比例圖。 |
| **彈出視窗**<br>點擊「總人數」(下) | **現掛(未看)人數趨勢** | 🟢 **[系統爬蟲擷取]**<br>**資料來源：** 網頁的**「狀態」**欄位。<br>**計算邏輯：** 狀態包含「現掛」即判定為尚未看診，遍歷時間區間繪製成折線圖。 |
| **彈出視窗**<br>點擊「區域分佈」 | **區域分佈** | 🟢 **[系統爬蟲擷取]**<br>**計算邏輯：** 透過 `classifyPatient()` 分區後，取用最新一筆快照，擷取急救室、A區、B區、留觀、兒科、婦產科的當下人數繪製甜甜圈圖。 |
| **彈出視窗**<br>點擊「滯留時間」 | **滯留時間** | 🟢 **[系統爬蟲擷取]**<br>**資料來源：** 網頁的**「掛號時間」**欄位。<br>**計算邏輯：** 計算非回診病患的 `(現在時間 - 掛號時間)`。若大於 24、48 或 72 小時，分別加總至對應變數，繪製成三條超過長期佔床時數的折線圖。 |
